Estructura básica de un HTML

Si quiero incluir código se utiliza la tag <code>. Aquí no la utilizo y pongo un <pre> dentro un <div>. Pongo estilo a este div y se aplica a todo el objeto correspondiente. 

                    <!DOCTYPE html>
                    <html lang="en">
                    <head>
                    <meta charset="UTF-8">
                    <meta name="viewport" content="width=device-width, initial-scale=1.0">
                    <title>Document</title>
                    </head>
                    <body>

                    </body>
                    </html>

Comentario escapado a continuación: <!-- comentario -->

Para comentar en CSS se utiliza: /* comentario */

CSS


            
El estilo de cada página de HTML se hace en CSS (siglas en inglés
                de Cascading Style Sheets), y se linkea al HTML mediante un
                <link rel="stylesheet"
                href="style.css">. En este archivo CSS se localiza
                el estilo de la página. Es clave hacer un buen archivo CSS para
                que quede bien nuestra página. 

            
Se ponen # para indicar ids, y .X para indicar clases. 

            
Crear nuevo archivo .css

            
Se enlaca con la tag link:css + Tab

Estilos


            
<i> para poner en italics

            
<b> para poner en negrita (bold)

            
<u> para poner en subrallado

            
Para hacer superposiciones en z-index se necesita tener
                display:flex

            
Estas tags estan depreciadas! NO USAR excepto en casos puntuales
                personales

            
Para cambiar el estilo de letra, usar "style" dentro de la
                etiqueta correspondiente, con atributos "font-"

            
Transiciones

            
Se pone el atributo "transition:xs;" al objeto en el que queramos

Comentarios varios


            
Primero elementos, segundo CSS y ultimo el navegador

            
Clases para muchos elementos, IDs son solo para un elemento!

            
span solo envuelve el contenido, no el body

            
Los comentarios en cada sitio del HTML son diferentes

            
Poner estilo a todos los objetos juntos con un punto sirve para
                borrar el estilo predeterminado de todos estos objetos. Este "."
                aplica su estilo a todos los objetos del HTML.

            
href y src son a veces intercambiables

            
.png para logos, ya que no tiene fondo; .gif no se utiliza

            
display = flex se pone en el
                contenedor, no en el contenido; activar flex-wrap: wrap para que
                salten de linea y no solo se chafen. Los atributos del
                display:flex van sobre el contenedor con elementos.

Canvas business model


            
El canvas business model es una herramienta muy positiva en caso
                de tener un proyecto de negocio. 

            
Estas son algunas de las piezas clave: 

            

            
    
                
  1. 
                    Key Partners: quién te va a ayudar
                    a tirar adelante este modelo.
                
    2. 
                
  2. 
                    Key Activities: qué vas a hacer de
                    manera principal en tu negocio.
                
    3. 
                
  3. 
                    Value Propositions: que ofreces tu
                    que no ofrece nadie más para tus clientes.
                
    4. 
                
  4. 
                    Customer Relationships: que tipo
                    de relación esperan cada uno de nuestros segmentos de
                    clientes.
                
    5. 
                
  5. 
                    Channels: como quieren nuestros
                    clientes que lleguemos a ellos.
                
    6. 
                
  6. 
                    Customer Segments: para quien
                    aportamos valor, cuales son nuestros clientes mas
                    importantes
                
    7. 
                
  7. 
                    Key Resources: cuales son los
                    recursos que demanda nuestras value proposition
                
    8. 
                
  8. 
                    Cost structure: cual es nuestro
                    coste inherente más importante en nuestro modelo de negocio?
                
    9. 
                
  9. 
                    Revenue Streams: por qué cosa
                    nuestros clientes estan realmente dispuestos a pagar?
